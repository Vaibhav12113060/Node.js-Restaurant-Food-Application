# 🍽️ Node.js Restaurant Food Application

A backend RESTful API for a Restaurant Food Application built using **Node.js**, **Express.js**, and **MongoDB**. This project focuses on scalable API design, authentication, and clean backend architecture for a real-world food ordering system.

🚧 **Live Deployment:** Not deployed yet

---

## 🚀 Features
- User authentication & authorization (JWT-based)
- Secure API access with protected routes
- CRUD operations for food items
- Restaurant menu management
- MongoDB schema design using Mongoose
- Structured RESTful API responses
- Centralized error handling
- Modular and scalable backend architecture

---

## 🛠️ Tech Stack
- Node.js
- Express.js
- MongoDB
- Mongoose
- JavaScript (ES6+)
- JWT Authentication
- Postman (API testing)
- Git & GitHub

---

## 🔐 Authentication
- User registration & login
- Password hashing
- JWT-based authentication
- Protected routes for authorized users

---

## 📂 API Capabilities
- User signup & login
- Add new food items
- Update existing food items
- Delete food items
- Fetch all food items
- Fetch single food item details

---

## 🧠 Project Summary
This project was built to gain hands-on experience with **backend development and real-world API workflows**. It demonstrates the implementation of authentication, database integration, and RESTful principles while maintaining clean code structure and scalability.

The application simulates a restaurant backend system and can be extended further for full-stack or production use cases.
